01 Problem & Motivation

A model handed a whole chart drowns in it.

A vision-language model reads the question, scans the entire image, and gets pulled toward bars, labels, gridlines and legends that have nothing to do with the answer. The link between the question and the one region that actually matters is never made explicit — so it guesses from everything at once.

Chart+ Question→ LLM→ Answer ✕
The naïve approach — answer straight from the full image. Baseline: 63.0%.
Question→ Locate region→ Focused evidence→ Answer ✓
Our idea — find the target first, then answer only from what matters.
02 Approach

Don't retrain everything — give the model tools.

A capable model already knows how to reason. What it lacks is clean, focused evidence. So we wrap a purpose-built localization model and OCR readers as tools, and let the model call them to pinpoint and measure exactly the parts of the chart a question needs.

STEP 01

Chart QA dataset

Curate chart images paired with natural-language questions across five chart families.

STEP 02

Categorize types

Separate charts into vertical, horizontal, two-color bar, pie and line — each read differently.

STEP 03

Element detection

Detect every component — bars, segments, points, slices — inside each chart image.

STEP 04

Central point

Reduce each component to a single center point: a clean, unambiguous coordinate target.

STEP 05

Expression generation

Pair each point with a referring expression.

STEP 06

Fine-tune the locator

Train a referring-expression model to map any expression to its point on the chart.

03 The Dataset
To teach a model to locate a referring expression on a chart, you need (expression → point) pairs. None existed — so we built the dataset from scratch.
Source

ChartQA as the image base

We start from ChartQA images. They carry every component's value and label — but not its position on the image. Positions are exactly what we need to create.

Detection

Grounded-SAM finds components

We detect each component and reduce it to a single center point rather than a bounding box — a box on a pie chart spills into neighboring slices, while a point stays clean.

Per-type extraction

Each chart family, its own method // bar2 + pie are hard

Bars give their centers easily. A two-color bar needs color-transition segmentation to split each colored segment apart; pie slices have no clean rectangle, so we locate each slice's center directly.

Matching

Hungarian assignment

Detected points are matched to the dataset's values and labels with the Hungarian algorithm — a global minimum-distance assignment that survives the many misleading numbers nearby, where naïve nearest-neighbor fails.

Verification

Manual pass, then expressions

A manual check over the full set pushes decomposition to near-perfect accuracy. Templates then turn each labeled point into a referring expression — yielding the final (center point, expression) pairs.

04 The Locator · Training

One model, two stages.

We fine-tune Qwen2.5-VL-7B-Instruct with LoRA into a referring-expression locator. Stage 1 trains on the full 90K set. After reviewing the results we saw some chart types lagging — so Stage 2 continues from those weights, oversampling the weak families with ~10K extra examples to rebalance the training set.

05 Inference · Tool Orchestration

The model drives; the tools measure.

At inference, a capable model runs as an agent. It infers the chart type, plans which points the question needs, and calls three tools to gather focused, numeric evidence — sanity-checking every tool output against what it actually sees.

locate_expression()
the fine-tuned locator

Takes one referring expression, returns a single point in normalized [0,1000] coordinates. Multi-point questions become several separate calls.

ocr_values()
read printed numbers

Scans the chart and returns every numeric label with its position. The located point is matched to the nearest sensible value — used when values are printed on the chart.

point_to_value()
reconstruct the axis

When a value isn't printed, it rebuilds the y-axis scale from OCR'd tick labels and interpolates the value at the located point's height.

Agent loop
  1. Infer the chart type from the image — the question never states it.
  2. Plan: decide which data points are needed and which tool reads each value.
  3. Call locate_expression — one point per target.
  4. Read the value with ocr_values or point_to_value, by chart-type rule.
  5. Sanity-check each value against the visible chart; adjust if implausible.
  6. Compute & answer — then stop calling tools.
A short visual lesson

Five visual few-shot examples — each an image, a question, a tool trace, a matching rule and a final answer — teach the agent how to recognize chart types and use the tools with discipline, before it ever sees the test chart.

Grading by meaning

A separate pass judges each answer against the reference by meaning, not formatting. Small format discrepancies never cost a point.
